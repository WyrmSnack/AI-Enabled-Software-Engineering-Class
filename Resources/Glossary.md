# Technical Glossary

Common terms in AI-enabled software engineering.

## A

**API (Application Programming Interface)**: A set of protocols and tools for building software applications.

**Authentication**: Verifying who a user is.

**Authorization**: Determining what a user can do.

**Agent (AI Agent)**: An autonomous AI system that can perform tasks independently.

## B

**Backend**: Server-side of an application.

**Branch (Git)**: A parallel version of code.

**Bug**: An error or flaw in software.

## C

**CI/CD (Continuous Integration/Continuous Deployment)**: Automated testing and deployment.

**Commit**: A snapshot of code changes.

**CRUD**: Create, Read, Update, Delete operations.

**CORS (Cross-Origin Resource Sharing)**: Mechanism for cross-domain requests.

## D

**Database**: Organized collection of data.

**Debugging**: Finding and fixing bugs.

**Deployment**: Making software available for use.

**Docker**: Containerization platform.

## E

**Endpoint**: A specific URL in an API.

**Error Handling**: Managing and responding to errors.

## F

**FastAPI**: Modern Python web framework.

**Frontend**: Client-side of an application.

**Full-Stack**: Both frontend and backend development.

## G

**Git**: Version control system.

**GitHub**: Code hosting platform.

**GitHub Copilot**: AI pair programming tool.

## H

**HTTP (Hypertext Transfer Protocol)**: Protocol for web communication.

**HTTPS**: Secure version of HTTP.

## I

**IDE (Integrated Development Environment)**: Software for writing code.

**Integration Test**: Testing component interactions.

## J

**JSON (JavaScript Object Notation)**: Data format.

**JWT (JSON Web Token)**: Authentication token format.

## L

**LLM (Large Language Model)**: AI model trained on text data.

**Logging**: Recording application events.

## M

**MCP (Model Context Protocol)**: Protocol for AI context management.

**Migration**: Database schema changes.

## O

**ORM (Object-Relational Mapping)**: Database abstraction layer.

**OWASP**: Organization focused on web security.

## P

**PR (Pull Request)**: Request to merge code changes.

**pytest**: Python testing framework.

## R

**React**: JavaScript library for UIs.

**REST (Representational State Transfer)**: API architectural style.

**Repository (Repo)**: Code storage location.

## S

**SDLC (Software Development Lifecycle)**: Process of developing software.

**SQL (Structured Query Language)**: Database query language.

**SQL Injection**: Security vulnerability.

## T

**Test**: Code that verifies functionality.

**Type Hint**: Python type annotations.

## U

**Unit Test**: Test for individual functions.

**URL (Uniform Resource Locator)**: Web address.

## V

**Version Control**: Tracking code changes.

**Vite**: Frontend build tool.

## X

**XSS (Cross-Site Scripting)**: Security vulnerability.
